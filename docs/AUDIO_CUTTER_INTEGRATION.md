## 🎯 Огляд функціонал

### Що це дає адвокатові?

**Автоматична видобудь критичних фрагментів** з судових записів за часовими мітками.

**Раніше (ручний процес):**
1. Прослухати весь запис (2-3 години)
2. Знайти критичні моменти
3. Вручну виробити MP3 нарізки (FFmpeg)
4. Обробити кожну нарізку окремо

**Тепер (автоматизовано):**
1. Список часових меток в `search_phrases.txt`
2. Клік на кнопку "Запустити"
3. **Автоматична нарізка + ШІ-аналіз кожного фрагменту**
4. Готові папки з аудіо, контекстом, аналізом

### Типові використання:

- 📋 **Витяг показань свідків на користь захисту**
- 🚫 **Виділення моментів, де прокурор нечесний**
- 📍 **Доведення нарушень процедури**
- 💬 **Цитування ключових фраз судді**

## 🔌 API документація

### Endpoint: `POST /api/cut-audio`

**Запуск нарізки аудіо по часовим мітках.**

#### Параметри

```json
{
  "folder_path": "C:\\cases\\case_123",
  "phrases_file": "search_phrases.txt",
  "api_key": "sk-ant-api03-..."
}
```

| Параметр | Тип | Обов'язковий | Опис |
|----------|-----|-------------|------|
| `folder_path` | `str` | ✅ Так | Абсолютний шлях до папки справи |
| `phrases_file` | `str` | ❌ Ні | Ім'я файлу в папці (за замовч: `search_phrases.txt`) |
| `api_key` | `str` | ❌ Ні | Anthropic API key для ШІ-аналізу |

#### Відповідь (успіх)

```json
{
  "task_id": "a1b2c3d4",
  "folder": "C:\\cases\\case_123",
  "phrases_file": "C:\\cases\\case_123\\search_phrases.txt"
}
```

#### Відповідь (помилка)

```json
{
  "detail": "Папка не знайдена: C:\\invalid\\path"
}
```

#### Приклад cURL

```bash
curl -X POST "http://localhost:8000/api/cut-audio" \
  -F "folder_path=C:\\my_cases\\case_123" \
  -F "phrases_file=search_phrases.txt" \
  -F "api_key=sk-ant-api03-..."
```

#### Приклад Python

```python
import requests

response = requests.post(
    "http://localhost:8000/api/cut-audio",
    data={
        "folder_path": "C:\\my_cases\\case_123",
        "phrases_file": "search_phrases.txt",
        "api_key": "sk-ant-api03-...",
    }
)

print(response.json())
# → {'task_id': 'a1b2c3d4', ...}
```

## 🏗️ Архітектура інтеграції

### Вхідні компоненти

```
┌─────────────────────────────────────────────────────────┐
│                    КОРИСТУВАЧ                           │
└────────────────────┬────────────────────────────────────┘
                     │
         ┌───────────┴────────────┐
         │                        │
    ┌────▼─────┐         ┌───────▼──────┐
    │  Веб-UI  │         │  Streamlit   │
    │(FastAPI) │         │   UI         │
    └────┬─────┘         └───────┬──────┘
         │                       │
         └───────────┬───────────┘
                     │
         ┌───────────▼──────────┐
         │   /api/cut-audio     │  ← FastAPI endpoint
         │   (main.py)          │
         └───────────┬──────────┘
                     │
         ┌───────────▼────────────────┐
         │  services.py               │
         │  start_audio_cutting()     │  ← Background thread
         │  _audio_cutting_worker()   │
         └───────────┬────────────────┘
                     │
         ┌───────────▼────────────────┐
         │  audio_cutter.py           │  ← Бізнес-логіка
         │ (production-grade)         │
         │                            │
         │  - parse timestamps        │
         │  - find audio files        │
         │  - cut segments (FFmpeg)   │
         │  - generate reports        │
         └───────────┬────────────────┘
                     │
    ┌────────────────┴────────────────┐
    │                                 │
┌───▼─────────┐             ┌────────▼──┐
│ User folder │             │ FFmpeg    │
│ case_123    │             │ (stream)  │
└───┬─────────┘             └───────────┘
    │
    └── _CourtDefense/
        └── 02_нарізки_за_мітками/
            ├── фраза_1__recording__min_10-30/
            │   ├── нарізка.mp3
            │   ├── ШІ_АНАЛІТИКА.txt
            │   └── фрагмент_контексту.txt
            └── фраза_2__recording__min_15-45/
                └── ...
```

### Технічні деталі

1. **UI отримує запит** → `POST /api/cut-audio`
2. **FastAPI валідує** → перевіряє папку, файл фраз
3. **Background thread** → запускає `start_audio_cutting()`
4. **Worker функція** → викликає `cut_audio_by_timestamps()`
5. **Audio Cutter** → обробляє всю логіку
6. **UI отримує status** → через polling `/status/{task_id}`
7. **Результати** → у папці `_CourtDefense/`

### Обробка помилок

```python
# Всі помилки перехоплюються в worker функції
try:
    result = cut_audio_by_timestamps(folder_path)
except Exception as e:
    _upd(tid, stage="error", progress=100, error=str(e))
    # UI показує помилку користувачу
```

### Приклад 3: Використання через API

```bash
#!/bin/bash

FOLDER="/path/to/case"
PHRASES_FILE="search_phrases.txt"

# Запустити нарізку
RESPONSE=$(curl -s -X POST "http://localhost:8000/api/cut-audio" \
  -F "folder_path=$FOLDER" \
  -F "phrases_file=$PHRASES_FILE")

TASK_ID=$(echo $RESPONSE | jq -r '.task_id')
echo "Task ID: $TASK_ID"

# Опитувати статус
while true; do
  STATUS=$(curl -s "http://localhost:8000/status/$TASK_ID")
  STAGE=$(echo $STATUS | jq -r '.stage')
  PROGRESS=$(echo $STATUS | jq -r '.progress')
  
  echo "[$PROGRESS%] $STAGE"
  
  if [ "$STAGE" = "completed" ] || [ "$STAGE" = "error" ]; then
    break
  fi
  
  sleep 2
done
```

## 🛠️ Розв'язання проблем

### Проблема 1: "Файл з фразами не знайдено"

**Причина:** Файл `search_phrases.txt` не існує в папці справи

**Рішення:**
1. Переконайтесь, що файл створений: `search_phrases.txt`
2. Файл повинен бути в **корені папки справи**
3. Перевірте орфографію та кодування (UTF-8 найкраще)

### Проблема 2: "FFmpeg не знайдено"

**Причина:** FFmpeg не встановлений або не в PATH

**Рішення:**
1. Завантажити FFmpeg: https://ffmpeg.org/download.html
2. **Варіант А:** Додати до системного PATH
3. **Варіант Б:** Розпакувати в `ffmpeg/bin/ffmpeg.exe` в корені проекту

### Проблема 3: "Помилка доступу до папки"

**Причина:** Недостатньо прав на читання файлів

**Рішення:**
1. Переконайтесь, що папка доступна для читання
2. На Windows: перевірте права доступу властивостями папки
3. На Linux/macOS: `chmod 755 /path/to/case`

### Проблема 4: "Нарізка не повернула результат"

**Причина:** У папці немає аудіофайлів або транскрипцій

**Рішення:**
1. Перевірте, що папка містить: `.mp3`, `.wav`, `.m4a` файли
2. Або транскрипції: `.json`, `.txt` файли з текстом
3. Запустіть `/scan-folder` щоб побачити що знайдено

## 📊 Статистика та Моніторинг

### Доступні метрики в `/status/{task_id}`

```json
{
  "stage": "completed",
  "progress": 100,
  "message": "✅ Готово! Нарізано 8 фрагментів",
  "label": "🎵 Нарізка: case_123",
  "logs": [
    "[10:45:32] 🔍 Сканую папку та парсю фрази",
    "[10:45:35] 📂 Запускаю pipeline нарізки з 8 фраз",
    "[10:46:12] ✅ Готово! Нарізано 8 фрагментів (0 помилок)"
  ],
  "error": null
}
```

### Як інтерпретувати `stage`

| Stage | Значення |
|-------|----------|
| `preparing` | Підготовка (сканування папки) |
| `processing` | Обробка в процесі |
| `completed` | Успішно завершено |
| `error` | Помилка — перевірте `error` поле |
| `cancelling` | Користувач натиснув "Скасувати" |

## 🔐 Безпека та Best Practices

### Валідація вхідних даних

```python
# ✅ Правильно: Перевіряємо шляхи
if not Path(folder_path).exists():
    raise HTTPException(400, "Папка не знайдена")

# ❌ Неправильно: Не перевіряємо
shutil.rmtree(user_input_path)  # Небезпечно!
```

### Обробка помилок

```python
# ✅ Правильно: Ловимо все
try:
    result = process(user_input)
except Exception as e:
    log_error(e)
    return {"error": str(e)}

# ❌ Неправильно: Дозволяємо падати
result = process(user_input)
```

### Управління правами доступу

- 🔒 Користувачі можуть обробляти **тільки локальні папки**
- 🔒 Немає доступу до файлів за межами папки справи
- 🔒 API key зберігається в переменной оточення, а не в БД
