# 🎵 Quick Start: Audio Cutting by Key Phrases

**Як автоматично видобути критичні фрагменти з судових записів**

---

## 📝 Крок 1: Створити файл зі списком фраз

Створіть файл `search_phrases.txt` (простий текстовий файл) з переліком фраз, які вас цікавлять.

**Приклад: `search_phrases.txt`**
```
ключова фраза
важливе свідчення
порушення процедури
неправильне рішення
протоколу не відповідає
незаконне рішення
# Коментарі ігноруються (починаються з #)
докази підтверджують
```

**Правила:**
- Одна фраза на строку
- Будь-яке кодування (UTF-8, Windows-1251, CP1252) — скрипт розбиратиме
- Коментарі починаються з `#`
- Пусті строки ігноруються
- Пошук **case-insensitive** (великі/малі літери не важливі)

## 🚀 Крок 2: Запустити скрипт

### Варіант A: Інтерактивний CLI (рекомендовано)

```powershell
python run_audio_cutter.py
```

**Скрипт запитатиме:**
```
[1] Введіть шлях до папки справи: D:\my_cases\case_123
[2] Введіть шлях до файлу зі списком фраз [за замовчуванням: search_phrases.txt]: 
    (просто натисніть Enter, якщо файл в тій же папці)
```

**Результат:**
```
[Конвеєр] Аналіз файлу: recording_1.json
  [Нарізка] 1:00 → нарізка.mp3
  [✓] Нарізано та проаналізовано мітку 1:00
  [Нарізка] 15:30 → нарізка.mp3
  [✓] Нарізано та проаналізовано мітку 15:30

[Успіх] Нових подій: 2, Пропущено дублікатів: 0
```

### Варіант B: Programmatic (для інтеграції)

```python
from webapp.audio_cutter import cut_audio_by_phrases

result = cut_audio_by_phrases(
    case_folder="/path/to/case",
    search_phrases_file="search_phrases.txt"
)

print(f"Оброблено: {result['processed']}")
print(f"Пропущено: {result['skipped']}")
print(f"Всього: {len(result['matches'])}")
```

## ⏩ Ключова особливість: Чекпоінти (Idempotency)

### Перший запуск
```
[Запуск] Шукаю 3 фраз(и)...
[Конвеєр] Аналіз файлу: recording.json
  [Нарізка] 1:00 → нарізка.mp3
  [✓] Нарізано та проаналізовано мітку 1:00
  [Нарізка] 15:30 → нарізка.mp3
  [✓] Нарізано та проаналізовано мітку 15:30

[Успіх] Нових подій: 2, Пропущено дублікатів: 0
```

### Другий запуск (нічого нового)
```
[Запуск] Шукаю 3 фраз(и)...
[Конвеєр] Аналіз файлу: recording.json
  [ЧЕКПОІНТ] Фрагмент вже існує, пропускаємо нарізку для: recording.json на 1:00
  [ЧЕКПОІНТ] Фрагмент вже існує, пропускаємо нарізку для: recording.json на 15:30

[Успіх] Нових подій: 0, Пропущено дублікатів: 2
```

**Що це означає:** 
- Якщо ви додасте нові фрази, скрипт обробить тільки їх
- Старі фрагменти не перетворюються заново
- Економія часу: 60 секунд на стари фрази = 0 секунд 🚀

## 🎯 Практичні приклади

### Приклад 1: Виділення всіх невідповідностей у протоколі

```powershell
# Створити search_phrases.txt з такими фразами:
недоліки протоколу
порушення процедури
незафіксовано
неправильно оформлено
невідповідне трактування
```

Результат: Автоматично витягнути всі епізоди, де обговорюються проблеми з протоколом.

### Приклад 2: Зібрати всі показання свідків на користь клієнта

```powershell
# Створити search_phrases.txt:
свідок підтверджує
спостерігав це особисто
я бачив
чітко бачив
це факт
мої слова повні
```

Результат: Вилучити всі фрази, де свідок засвідчує факти на користь захисту.

### Приклад 3: Знайти всі моменти, де прокурор робить неправомірні висновки

```powershell
# Створити search_phrases.txt:
припускаю що
виходячи з припущення
можливо це означає
можливо що
на мою думку
```

Результат: Виділити спекулятивні вислови прокурора без доказів.

## 🔧 Налаштування (опціонально)

### Змінити тривалість нарізаного фрагмента

За замовчуванням кожна нарізка — це 60 секунд із центром на знайденій фразі.

Щоб змінити, відредагуйте `webapp/audio_cutter.py` рядок ~312:
```python
_cut_audio_segment(audio_file, output_audio, start_sec, duration_sec=90)  # 90 сек замість 60
```

### Змінити розмір контексту

За замовчуванням контекст — це ±15 секунд від фрази.

Щоб змінити, відредагуйте `webapp/audio_cutter.py` рядок ~256:
```python
matches = _search_phrase_in_text(text, phrase, context_seconds=30)  # 30 сек контексту
```

## ❌ Решення проблем

### Проблема 1: "Файл не знайдено"
```
[Помилка] Файл із фразами не знайдено!
Перевірений шлях: D:\my_case\search_phrases.txt
```

**Рішення:**
- Переконайтесь, що файл `search_phrases.txt` існує
- Переконайтесь, що шлях написано правильно (без опечаток)
- Якщо файл в іншій папці, вкажіть повний шлях

### Проблема 2: "FFmpeg не знайдено"
```
[Помилка] Помилка ініціалізації медіа-двигуна. Компонент FFmpeg не знайдено.
```

**Рішення:**
- Завантажте FFmpeg: https://ffmpeg.org/download.html
- Розпакуйте в папку `ffmpeg/bin/` в корені проекту
- Або встановіть FFmpeg в системі (чи додайте до PATH)

### Проблема 3: "Фраз не знайдено у транскрипціях"
```
[Інформація] Фраз не знайдено у транскрипціях.
```

**Рішення:**
- Перевірте, що транскрипції присутні (файли .json або .txt)
- Перевірте, що фрази дійсно присутні в записі
- Перевірте орфографію у search_phrases.txt

### Проблема 4: "Кодування читання"
Якщо скрипт раніше падав на неправильному кодуванні файлу...

**Рішення:**
- ✅ Тепер це не буває! Скрипт автоматично пробує UTF-8 → CP1251 → CP1252 → Latin1 → ASCII

## 💡 Поради

1. **Почніть з кількох фраз** — спочатку 3-5, потім розширяйте
2. **Коротші фрази — краще** — "ключова фраза" краще за "дуже довга ключова фраза що навіть не буває"
3. **Регулярно оновлюйте список** — додавайте нові фрази, коли знаходите нові докази
4. **Перегляньте контекст** — переконайтесь, що контекст дійсно підтримує вашу позицію
5. **Збережіть чекпоінти** — не видаляйте папку `_CourtDefense`, вона зберігає історію
