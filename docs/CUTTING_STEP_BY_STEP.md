# 🎵 Пошаговая инструкция по нарізці аудіо | Step-by-Step Guide

## 🎯 Загальна схема

```
1. Создать файл search_phrases.txt
            ↓
2. Вказати папку справи
            ↓
3. Нажати кнопку "Запустити"
            ↓
4. Система автоматично:
   • Читає файл з фразами
   • Шукає фрази в транскрипціях
   • Різає аудіо в потрібних місцях
   • Генерує аналіз для кожної нарізки
            ↓
5. Результати готові в папці _CourtDefense/
```

## 📋 КРОК 1: Подготовка файла фраз

### Шаг 1.1: Створити текстовий файл

1. Відкрити **Notepad** (блокнот)
2. Скопіювати текст (див. нижче)
3. Зберегти як `search_phrases.txt` в папці справи

```

2 прокурор припускає без доказів
15:20---15:35

3 суддя не дозволив мовити
22:10---22:25
```

### Шаг 1.3: Де зберегти файл

```
D:\cases\case_123\                    ← Ваша папка справи
├── Отсортированные_данные_для_суда/  ← Аудіофайли

```

**ВАЖЛИВО:** Файл ПОВИНЕН бути в папці справи, не в підпапці!

## ⏱️ КРОК 2: Знайти правильні часові мітки

### Як дізнатися час фрази?

#### Вариант А: За допомогою VLC (MediaPlayer)

1. Відкрити аудіофайл в **VLC**
2. Слухайте запис
3. Коли почнеться цікава фраза → запам'ятайте час внизу
4. Коли закінчиться фраза → запам'ятайте час

**Приклад VLC:**
```
[████░░░░░░░] 00:10:30 / 01:15:00
               ↑
             ЦЕЙ ЧАС!
```

#### Вариант Б: За допомогою JSON транскрипції

Якщо у вас є файл `recording_1.json` з Whisper:

```json
{
  "segments": [
    {
      "id": 5,
      "start": 630.45,      ← Це = 00:10:30
      "end": 645.75,        ← Це = 00:10:45
      "text": "батько каже..."
    }
  ]
}
```

**Як конвертити:**
- 630.45 сек = 630 / 60 = 10 хвилин 30 секунд = 10:30
- 645.75 сек = 645 / 60 = 10 хвилин 45 секунд = 10:45

#### Вариант В: Запитайте ChatGPT або мене!

```
"На якому часу свідок каже про алібі?"
→ Я допоможу знайти правильний час
```

## ❌ КРОК 6: Розв'язання проблем

### Проблема 1: "Файл search_phrases.txt не знайдено"

**Причина:** Файл не в правильному місці або неправильна назва

**Рішення:**
```
1. ✓ Файл у папці справи (не в підпапці)
2. ✓ Точна назва: search_phrases.txt
3. ✓ Кодування: UTF-8
4. ✓ Розширення: .txt (не .doc, не .docx)
```

**Перевіріть:**
```
D:\cases\case_123\search_phrases.txt  ← ПРАВИЛЬНО
D:\cases\case_123\data\search_phrases.txt  ← НЕПРАВИЛЬНО
D:\cases\case_123\search_phrases.doc  ← НЕПРАВИЛЬНО
```

### Проблема 2: "Фраза не знайдена"

**Причина:** Фраза не в транскрипціях або час неправильний

**Рішення:**
```
1. Перевірте орфографію в файлі фраз
2. Перевірте, що в папці справи є .json або .txt файли
3. Переконайтесь у правильності часу
4. Спробуйте скопіювати фразу ПРЯМО З ТРАНСКРИПЦІЇ
```

### Проблема 3: "FFmpeg не знайдено"

**Причина:** FFmpeg не встановлений

**Рішення:**
```
1. Скачайте FFmpeg: https://ffmpeg.org/download.html
2. Розпакуйте в папку:
   Project/ffmpeg/bin/ffmpeg.exe
3. Або додайте до системного PATH
4. Перезавантажте комп'ютер
```

### Проблема 4: "Час неправильний"

**Причина:** Формат часу не MM:SS---MM:SS

**Рішення:**
```
❌ НЕПРАВИЛЬНО:         ✓ ПРАВИЛЬНО:
10.30---10.45          10:30---10:45
10:30-10:45            10:30---10:45
10:30 - 10:45          10:30---10:45
1:30---1:45            01:30---01:45
```

## 💡 ПОРАДИ

### 1. Кількість фраз

```
❌ Замало: 1-2 фрази
  → Буде занадто мало доказів

✓ Оптимально: 5-15 фраз
  → Охоплює основні моменти справи

⚠️ Занадто: 50+ фраз
  → Перевантажить адвоката
```

### 2. Довжина нарізки

```
за замовчуванням: 15 сек до фрази + 15 сек після
(разом = 30 сек + сама фраза)

Якщо потрібна коротша нарізка:
10:30---10:35  ← тільки 5 сек
```

### 3. Вибір фраз

```
✓ ДОБРО:
  • "свідок каже, що видів це особисто"
  • "експертиза підтверджує"
  • "суддя не дав говорити"

❌ ПОГАНО:
  • "значить" або "типу" (занадто коротко)
  • "він каже" (занадто спільна фраза)
  • "е-е-е" (шум, не доказ)
```
